Quality Assurance

Acceptance criteria, test scenarios, edge cases, and untestable behaviours.

Acceptance Criteria
Given a user navigates to the registration page, When they provide a unique email, a valid full name, a role, and a company name, Then their account is successfully created, and they are automatically logged in.
Given a a new user navigates to the registration page, When they attempt to register without providing a 'Company Name', Then the system prevents registration and displays a clear error message indicating that 'Company Name' is required.
Given a new user navigates to the registration page, When they attempt to register with an email address that is already in use, Then the system prevents registration and displays a clear error message indicating the email is taken.
Given a user navigates to the login page, When they select 'Sign in with Google' and successfully authenticate via their Google account, Then the user is successfully logged into the application.
Given a user navigates to the login page, When they enter valid email and password credentials, Then the user is successfully logged into the application.
Given a user is successfully logged in, When 24 hours have elapsed since their last login or activity, Then their session is automatically invalidated, and subsequent actions require re-authentication.
Given a user is logged in and navigates to the project dashboard, Then the dashboard page loads successfully and prominently displays a section for 'Active User Metrics'.
Given a user is logged in and navigates to the project dashboard, Then the dashboard page loads successfully and prominently displays a section for 'System Status'.
Given a user is logged in and navigates to the project dashboard, Then the dashboard page loads successfully and prominently displays a section for 'Project Lists'.
Test Scenarios
Verify successful new user registration using unique email, full name, role, and company name, followed by automatic login.
Verify user login using Google authentication, including redirection to Google and successful return to the application.
Verify user login using email and password for a pre-registered account.
Verify that attempting to register with an email address already associated with an account results in an appropriate error message and prevents registration.
Verify that attempting to register with a blank 'Company Name' field results in an error message and prevents registration.
Verify that attempting to register with any other required field (Email, Full Name, Role) left blank results in an error message and prevents registration.
Verify that attempting to log in with an incorrect email or password results in an error message and prevents login.
Verify that after a user logs in, their session persists for 24 hours, and then expires, requiring re-login.
Verify that the project dashboard loads correctly and displays distinct sections for 'Active User Metrics', 'System Status', and 'Project Lists' without content errors.
Edge Cases
Test user registration and login with email addresses, full names, roles, and company names containing maximum allowed characters.
Test user registration and login with email addresses, full names, roles, and company names containing special characters (e.g., accented letters, symbols permitted by Firebase/PostgreSQL).
Test concurrent logins for the same user account from multiple browsers or devices to observe session management behavior.
Test user registration with malformed email addresses (e.g., missing '@', missing domain, multiple '@' symbols).
Test login attempts with a valid email but an empty or excessively long password.
Observe dashboard load times and responsiveness when displaying a very large number of projects in the 'Project Lists' section.
