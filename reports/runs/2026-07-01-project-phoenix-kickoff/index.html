Quality Assurance

Acceptance criteria, test scenarios, edge cases, and untestable behaviours.

Acceptance Criteria
Given a user is on the registration page, When they fill in a valid Email, Full Name, Role, and Company Name, and submit, Then their account is successfully created, they are logged in, and redirected to the project dashboard.
Given a user is on the registration page, When they attempt to register without providing an Email, Then an error message is displayed indicating the Email field is required, and registration is prevented.
Given a user is on the registration page, When they attempt to register without providing a Full Name, Then an error message is displayed indicating the Full Name field is required, and registration is prevented.
Given a user is on the registration page, When they attempt to register without providing a Role, Then an error message is displayed indicating the Role field is required, and registration is prevented.
Given a user is on the registration page, When they attempt to register without providing a Company Name, Then an error message is displayed indicating the Company Name field is required, and registration is prevented.
Given a user is on the login page, When they provide valid email/password credentials, Then they are successfully logged in and redirected to the project dashboard.
Given a user is on the login page, When they click the "Sign in with Google" option, Then they are successfully authenticated via Google and redirected to the project dashboard.
Given a user is logged in, When 24 hours pass since their last successful authentication, Then their session token expires, and subsequent attempts to access protected resources should require re-authentication.
Given a user is on the registration page, When they attempt to provide an address or telephone number, Then these fields are not collected or stored by the system.
Given a user is logged in and navigates to the project dashboard, Then the dashboard displays the active user metrics.
Given a user is logged in and navigates to the project dashboard, Then the dashboard displays the current system status.
Given a user is logged in and navigates to the project dashboard, Then the dashboard displays lists of projects.
Given the backend services are running, Then user session information is stored in the PostgreSQL database.
Test Scenarios
**User Registration - Happy Path:** Successfully register a new user using email/password with valid Email, Full Name, Role, and Company Name.
**User Registration - Google Sign-in Happy Path:** Successfully register and log in a new user using Google sign-in.
**User Login - Happy Path:** Log in successfully with valid email/password credentials.
**User Login - Google Sign-in Happy Path:** Log in successfully with Google account credentials.
**Dashboard Access - Valid Session:** Log in and verify the project dashboard loads correctly, displaying active user metrics, system status, and project lists.
**User Registration - Invalid Email:** Attempt to register with an incorrectly formatted email address.
**User Registration - Existing Email:** Attempt to register with an email address already associated with an account.
**User Registration - Missing Required Fields:** Attempt to register by omitting Email, Full Name, Role, or Company Name individually and in combination.
**User Login - Incorrect Credentials:** Attempt to log in with incorrect email or password.
**User Login - Non-existent Account:** Attempt to log in with an email that is not registered.
**Session Expiration:** Log in, wait for 24 hours (or simulate session expiry), then attempt to access a protected resource and confirm re-authentication is required.
**Unauthorized Access:** Attempt to access authenticated dashboard features without being logged in or with an expired session.
Edge Cases
**User Registration - Max Length Inputs:** Test registration with maximum allowed character lengths for Email, Full Name, Role, and Company Name fields (if limits are defined).
**User Registration - Special Characters:** Test registration with Full Name, Role, and Company Name fields containing special characters, symbols, and Unicode characters.
**Concurrent Registrations:** Attempt multiple user registrations simultaneously with identical or similar data.
**Concurrent Logins:** Log in to the same user account from multiple devices or browser tabs simultaneously.
**Session Expiration Handling:** While actively performing actions in the application, simulate session expiration and observe the user experience (e.g., auto-logout, error message on next action, graceful redirection).
**Empty Dashboard Data:** What does the dashboard display if there are no active users, no system status information, or no projects?
