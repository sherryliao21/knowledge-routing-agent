The team is on track to deliver a core version of the Phoenix Project by September 1st, which is critical for supporting the planned marketing campaign launch shortly thereafter. Quality Assurance

Acceptance criteria, test scenarios, edge cases, and untestable behaviours.

Acceptance Criteria
Given a user attempts to sign in using Google, When the sign-in is successful, Then the user is authenticated and redirected to the project dashboard.
Given a user attempts to sign in using their email and password, When the sign-in is successful, Then the user is authenticated and redirected to the project dashboard.
Given an authenticated user, When 24 hours have passed since their last token refresh or initial login, Then their session token expires and they are required to log in again.
Given a user is on the registration page, When they provide a valid email address, full name, role, and company name, Then their account is successfully created.
Given a user is on the registration page, When they are completing the registration form, Then fields for address and telephone number are not presented or required.
Given an authenticated user navigates to the dashboard, Then the dashboard displays a section dedicated to active user metrics.
Given an authenticated user navigates to the dashboard, Then the dashboard displays a section dedicated to system status.
Given an authenticated user navigates to the dashboard, Then the dashboard displays a list of projects relevant to the user's account.
Given typical user load and common user actions (e.g., login, dashboard view), When these actions are performed, Then the system response times are consistently fast and meet defined performance thresholds.
Given an attempt is made to establish a connection to the PostgreSQL database from a public internet address, When the connection attempt is initiated, Then the connection is rejected.
Test Scenarios
Verify successful user login using a valid Google account.
Verify successful user login using a registered email and correct password.
Verify login failure with an unregistered email address.
Verify login failure with a registered email and incorrect password.
Verify that an authenticated user is automatically logged out if 24 hours pass without re-authentication or token refresh.
Verify successful user registration when all required fields (email, full name, role, company name) are provided.
Verify user registration fails when any required field (email, full name, role, company name) is missing.
Verify that the user registration form does not include input fields for address or telephone number.
Verify that the project dashboard loads successfully for an authenticated user.
Verify that the 'Active User Metrics' section is visible and populated with data on the dashboard.
Verify that the 'System Status' section is visible and displays relevant information on the dashboard.
Verify that the 'Project Lists' section is visible and displays available projects on the dashboard.
Edge Cases
Attempting Google sign-in with a Google account that is not associated with any valid user profile in the system.
Executing concurrent login attempts for the same user account from multiple different devices or browsers.
Attempting to perform authenticated actions using an expired or manually revoked session token.
Rapidly performing consecutive login and logout operations.
Attempting to register with an email address that is already registered in the system.
Attempting to register with an email address in an invalid format (e.g., missing '@', invalid domain).
Submitting unusually long strings for 'Full Name', 'Role', or 'Company Name' during registration.
Inputting special characters or malicious script attempts into any text fields during registration.
Loading the dashboard when there are no active users recorded in the system.
Loading the dashboard when the system status is reported as an error or unavailable.
Loading the dashboard when no projects are associated with the authenticated user's account.
Testing dashboard performance with a very large number of projects or extensive metric data to display.
