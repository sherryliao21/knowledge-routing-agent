Acceptance Criteria
Given the system is deployed, When a new user attempts to register with Email, Full Name, Role, and Company Name, Then the registration shall be successful, and the user shall be authenticated.
Given an unauthenticated user, When they attempt to sign in using their Google account, Then they shall be successfully authenticated and granted access.

Given an authenticated user, When their session token has reached 24 hours since issuance, Then their session shall expire, requiring re-authentication.
Given an authenticated user, When they navigate to the Project Dashboard, Then the dashboard shall display metrics related to active users.
Given an authenticated user, When they navigate to the Project Dashboard, Then the dashboard shall display the current system status.
Given an authenticated user, When they navigate to the Project Dashboard, Then the dashboard shall display lists of projects.
Given the PostgreSQL database is deployed, When attempting to access the database from the public internet, Then all connection attempts shall be rejected.
Given the Cloud Run service for the backend, When it attempts to connect to the PostgreSQL database, Then the connection shall be established exclusively via Serverless VPC Access.
Given any critical user action (e.g., login, dashboard load), When the action is performed, Then the system response time must be 'super fast' (per D008).
