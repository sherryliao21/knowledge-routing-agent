Quality Assurance

Acceptance criteria, test scenarios, edge cases, and untestable behaviours.

Acceptance Criteria (11)
Given a JWT access token is issued, When the current time exceeds 24 hours from issuance, Then the JWT access token must be considered invalid.
Given a user attempts a password reset, When the user has made 3 or fewer password reset requests within a 60-minute window, Then a password reset email is successfully sent for each valid request.
Given a user attempts a password reset, When the user has made more than 3 password reset requests within a 60-minute window, Then subsequent requests within that window must be rejected, and no further password reset email is sent.
Given a user makes API requests, When the user makes 100 or fewer requests within a 60-second window, Then all requests are processed successfully.
Given a user makes API requests, When the user makes more than 100 requests within a 60-second window, Then subsequent requests from that user within that window must be throttled or rejected.
Given valid user credentials, When a user attempts to log in, Then a valid JWT access token and refresh token are issued, and a successful login response is returned.
Given invalid user credentials, When a user attempts to log in, Then an error response indicating invalid credentials is returned, and no tokens are issued.
Given a valid and non-expired refresh token, When a user requests to refresh their tokens, Then a new valid JWT access token and refresh token are issued, and a successful response is returned.
Given an invalid or expired refresh token, When a user requests to refresh their tokens, Then an error response is returned, and no new tokens are issued.
Given a request to initiate a password reset for a registered email address, When the password reset rate limit is not exceeded, Then a password reset email is sent, and a generic success message is returned to the user.
Given a request to initiate a password reset for an unregistered email address, When the password reset rate limit is not exceeded, Then a generic success message is returned to the user, and no password reset email is sent.

Attempt to log in with a correct username and an incorrect password, verifying authentication fails with an appropriate error.
Attempt to log in with an unregistered username, verifying authentication fails with an appropriate error (e.g., invalid credentials, not user not found).
Use a valid, non-expired refresh token to obtain a new pair of access and refresh tokens, verifying the new tokens are issued.
Use an expired refresh token to obtain new tokens, verifying the refresh request fails with an appropriate error.
Initiate a password reset for a valid user email and confirm the password reset email is sent (observing D004 rate limit).
Initiate a password reset for an email not registered in the system, confirming a generic success message is returned but no email is sent.
Edge Cases (7)
Test JWT token validity exactly at the 24-hour mark (considering potential time synchronization differences).
Test concurrent password reset requests from the same user across multiple sessions or devices.
Test password reset requests made exactly at the 60-minute boundary to reset the rate limit.
Test API Gateway throttling with a burst of 100 requests in a very short period (e.g., 1 second).
Test API Gateway throttling for a user making 100 requests in the first 5 seconds of a minute, then pausing until the next minute.
Test login attempts with special characters or maximum length values for username/password fields.
Test refresh token usage immediately prior to its natural expiry.
