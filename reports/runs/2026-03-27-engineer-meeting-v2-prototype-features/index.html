Stakeholder Executive Briefing

Business-focused summary: impact, alignment decisions, and top open questions.

Business Summary

Our initial product launch (V1) will focus on two core features: allowing users to upload photos and then process those photos into 3D models. Users will access the system seamlessly through our existing external login portal, ensuring a consistent and secure experience without needing a new account or password. We've also outlined several advanced features for future releases to enhance collaboration and user control.

Key Business Outcomes
Users will have a simple, secure login experience by using our existing company portal.
The product will deliver immediate value by enabling users to upload photos and generate 3D models, addressing key business needs.
The system is designed for future growth, with plans to introduce collaboration tools and advanced user controls in later versions.
User progress during modeling will be clearly visible, improving satisfaction and transparency.
Initial focus on a 'Normal Mode' ensures ease of use for the majority of users, with advanced customization (Expert Mode) planned for later.

Implementation Scope

Constraints

Album access must be restricted to the owner only; multi-user access for albums is not permitted. D011
Architectural Decisions

For V1, restrict output modeling functionality to 'Normal Mode' with default parameters only, deferring 'Expert Mode' to future versions. D016

Quality Assurance

Acceptance criteria, test scenarios, edge cases, and untestable behaviours.

Acceptance Criteria
Given a user attempts to log in, When they are redirected to the external SSO Portal, Then successful authentication on the external Portal grants access to the system.
The system's user interface must not contain any internal login forms or registration links.
The system must allow users to upload photos.
The system must allow users to initiate output modeling.
The sidebar interface must visually distinguish between 'Album' and 'Project' folder types.
The sidebar interface must allow separate management of photos within 'Album' folders and modeling data within 'Project' folders.
Given a user is in an 'Album' folder, When they select photos to add, Then the selected photos are successfully added to the 'Album'.
Given a user is in an 'Album' folder, When they select one or more photos and choose to delete them, Then the selected photos are successfully removed from the 'Album'.
Given a user is in an 'Album' folder, When they use the search functionality for photos, Then the system displays photos matching the search criteria.
Given a modeling process is initiated, When the modeling process starts, Then a progress bar is displayed showing the status of each stage (e.g., AT computing, point cloud generation, model generation).
Given a user is creating a new 'Project', When they access the photo selection interface, Then they can view and select photos from their available photo list.
Given a user has selected photos during 'Project' creation, When they confirm the selection, Then the chosen photos are successfully imported into the new 'Project' folder.
Given an 'Album' is created by a specific user, When another user attempts to view or access that 'Album', Then access is denied.
The system must only present options for 'Normal Mode' modeling (default parameters) and not display any 'Expert Mode' adjustable parameters for V1.

The system SHALL support the generation of output models (D005).

The system SHALL allow users to delete photos from an Album (D007).
The system SHALL allow users to search for photos within an Album (D007).
The system SHALL display the progress of modeling tasks via a progress bar (D008).
The modeling progress bar SHALL display the status of each stage of the modeling process, including AT computing, point cloud generation, and model generation (D008).
The system SHALL allow users to select existing photos from their photo list for import into a Project folder during project creation (D009).

The system SHALL support future functionality to allow designating other users to jointly view modeling results in a project folder (D010).
The system SHALL support future functionality for resumable uploads where upload progress is preserved via a caching mechanism if an upload is interrupted (D012).
The system SHALL support future functionality to display used and remaining storage on the user's personal page, and dynamically adjust user photo-upload limits based on system settings and permissions (D013).
The system SHALL support future functionality to provide a mechanism to adjust parameters and re-run AT computation when it fails (D014).

Data Flows & Integrations
User authentication requests SHALL be redirected from this system to the external Portal's SSO system, which SHALL return an authentication token/session to this system (D001).
User-uploaded photo data SHALL flow from the client application to the system's storage component (D004).
Selected photo metadata or references from 'Album' storage SHALL be associated with or copied into 'Project' folders during project creation (D009).
Project photo data SHALL flow from storage to a modeling processing component for output modeling (D005).
Modeling progress updates (e.g., AT computing status, point cloud generation status, model generation status) SHALL flow from the modeling processing component to the system's user interface (D008).
Generated model output data SHALL flow from the modeling processing component to 'Project' storage (D005).
Interface Contracts
SSO Integration (D001): The system SHALL integrate with the external Portal's SSO service. This implies an authentication redirect/callback mechanism. Expected input: Authentication token or session identifier from the SSO provider. Expected output: An authenticated user session within this system.
Photo Upload API (D004, D007): The system SHALL expose an endpoint for uploading photo files, likely supporting multipart/form-data. This API SHALL accept parameters for associating photos with an 'Album' (e.g., POST /api/albums/{albumId}/photos).
Photo Management API (D007): The system SHALL expose APIs for deleting photos (e.g., DELETE /api/albums/{albumId}/photos/{photoId}) and searching photos within an Album (e.g., GET /api/albums/{albumId}/photos?search={query}).
Project Creation API (D009): The system SHALL expose an API for creating new projects, allowing users to specify project details and provide a list of existing photo identifiers from their Albums to be imported (e.g., POST /api/projects, with payload including photo_ids).
Modeling Process API (D005, D008, D016): The system SHALL expose an API to initiate the modeling process for a given project (e.g., POST /api/projects/{projectId}/model/start). The system SHALL also expose an API to query the status of an ongoing modeling process, returning current stage and progress (e.g., GET /api/projects/{projectId}/model/status).
Folder Management API (D006): The system SHALL expose an API to retrieve a list of user's 'Album' and 'Project' folders (e.g., GET /api/folders).
