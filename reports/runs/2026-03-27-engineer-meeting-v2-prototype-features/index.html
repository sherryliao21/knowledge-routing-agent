Stakeholder Executive Briefing

Business-focused summary: impact, alignment decisions, and top open questions.

Business Summary

The team has finalized the core features for the initial version of the new system, prioritizing user access through the existing company login, photo uploads, and 3D modeling capabilities. This plan ensures a streamlined user experience and quicker delivery of fundamental functionality. More advanced features, such as collaborative project sharing and expert modeling controls, are slated for future development phases.

Key Business Outcomes
Users will access the system seamlessly using their existing company login, eliminating the need for new credentials.
The first version of the product will launch faster by focusing on the essential features of photo upload and generating 3D models.
Users will benefit from an organized workflow, allowing them to easily manage photos and modeling projects, and transparently track the progress of their modeling tasks.
The initial user experience will be simpler, as the first version will use default modeling settings, making it easier for new users to get started.

Quality Assurance

Acceptance criteria, test scenarios, edge cases, and untestable behaviours.

Acceptance Criteria
Given a user attempts to access the system, When they are redirected to the external SSO Portal, Then successful authentication on the SSO Portal grants them access to the system.
Given a user is on the system's login page, When they attempt to find an internal login form or registration link, Then no internal login or registration functionality is displayed or accessible.
Given a user is logged into the system, When they view the sidebar, Then 'Album' type folders are visually distinguishable from 'Project' type folders.
Given a user navigates to an 'Album' folder, Then only photo-related content and actions are displayed.
Given a user navigates to a 'Project' folder, Then only modeling data-related content and actions are displayed.
Given a user is in an 'Album' folder, When they upload a supported photo file, Then the photo is added to the album.
Given a user has selected one or more photos in an 'Album', When they initiate the delete action, Then the selected photos are removed from the album.
Given a user inputs a search query into the photo search functionality within an 'Album', When they execute the search, Then only photos matching the query are displayed.
Given a modeling process has started, When the user views the modeling status, Then a progress bar is displayed.
Given a modeling process is underway, When a specific stage (e.g., AT computing, point cloud generation, model generation) is active, Then the progress bar accurately reflects the status of that stage.
Given a modeling process is complete, Then the progress bar indicates 100% completion or a 'Completed' status.
Given a user initiates the creation of a new 'Project', When presented with a photo selection interface, Then they can browse and select photos from their existing 'Album' photo list.
Given a user has selected photos during project creation, When they confirm the project creation, Then the selected photos are imported and associated with the new 'Project' folder.
Given a user has uploaded photos and is proceeding to modeling, When the modeling parameters are presented, Then only 'Normal Mode' (default parameters) options are available.
Given a user is in the modeling workflow, When they attempt to access 'Expert Mode' parameter adjustments, Then no 'Expert Mode' functionality is accessible.
