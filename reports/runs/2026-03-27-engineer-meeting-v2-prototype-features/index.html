Stakeholder Executive Briefing

Business-focused summary: impact, alignment decisions, and top open questions.

Business Summary

Our new system's initial version (V1) will focus on two core capabilities: managing user photos and enabling 3D model creation from those photos. Users will log in exclusively through our existing external portal, streamlining access and avoiding the need for a separate login system. We are prioritizing these essential features for a faster first release, with more advanced functionalities planned for future updates.

Key Business Outcomes
Users will experience a simple and secure login process by using our established external portal.
The initial product provides immediate value by allowing users to upload, manage, and process photos into 3D models.
User workflows are streamlined by clearly separating personal photo collections ('Albums') from modeling projects ('Projects').
Users will be kept informed with detailed progress updates as their 3D models are being generated.
By focusing on core features for V1, we can launch sooner and gather valuable user feedback to guide future enhancements.

Quality Assurance

Acceptance Criteria
Given a user attempts to log in, When they initiate the login process, Then they are redirected to the external SSO Portal for authentication (D001).
Given a user attempts to log in, When authentication is successful via the external SSO Portal, Then they are successfully logged into the system (D001).
Given a user attempts to register or log in using internal credentials, When they interact with the system's interface, Then no internal login or registration functionality is presented (D002).
Given a user views the sidebar, Then 'Album' folders and 'Project' folders are visually distinct from each other (D005).
Given a user views an 'Album' folder, Then its content area is dedicated to managing photos (D006).
Given a user views a 'Project' folder, Then its content area is dedicated to managing modeling data (D006).
Given a user is in an 'Album', When they attempt to add one or more photos, Then the photos are successfully added to the album (D007).
Given a user is in an 'Album' containing photos, When they select a photo and initiate deletion, Then the photo is successfully removed from the album (D008).
Given a user is in an 'Album' containing photos, When they use the search functionality with a valid query, Then only photos matching the search criteria are displayed (D009).
Given an output modeling process is initiated, When the process is active, Then a visual progress bar is displayed to the user (D010).
Given an output modeling process is active, When the progress bar is displayed, Then it shows distinct sub-stages: AT computing, point cloud generation, and model generation (D011).
Given a user is creating a new project, When they are prompted to add photos, Then they can browse and select photos from their existing 'Album' photo list (D012).
Given a user selects photos from an 'Album' during project creation, When they confirm the selection, Then the selected photos are imported into the new project folder (D012).
Given a user owns an 'Album', When another user attempts to access that 'Album', Then the other user is denied access (D014).
Given a user attempts to share an 'Album' with other users, Then no multi-user access or sharing options are available for 'Album' folders (D015).
Given a user initiates a modeling process in V1, When the system prepares for computation, Then it automatically uses default parameters without requiring user adjustment (D020).
Given a user has uploaded photos for modeling in V1, When they look for advanced parameter settings, Then no 'Expert Mode' or adjustable modeling parameters are available (D020).
Test Scenarios
**Successful SSO Login:** Navigate to the system, verify redirection to the external SSO portal, successfully log in, and confirm landing on the expected system dashboard.
**Attempt Internal Login/Registration:** Verify that no login or registration forms/buttons exist within the application interface itself, confirming exclusive reliance on SSO.
**Create New Album and Add Photos:** Create a new 'Album' folder. Upload a single photo successfully. Upload multiple photos simultaneously and verify all are added.
**Delete Photos from Album:** Add several photos to an album. Select and delete a single photo. Select multiple photos and delete them at once. Verify photos are removed.
**Search Photos in Album:** Add photos with distinct names/tags to an album. Perform a search using a full name, a partial name, and a non-existent name. Verify accurate search results.
**Verify Album Privacy:** Create an album as User A. Attempt to access User A's album (e.g., via direct link if available, or through shared UI elements) while logged in as User B. Confirm access is denied.
**Initiate Modeling with Progress Bar:** Create a 'Project' and import photos. Initiate a modeling job. Observe the visual progress bar, verifying it appears and displays distinct stages (AT computing, point cloud generation, model generation) updating correctly.
**Create New Project and Import Existing Photos:** Create a new 'Project'. During project creation, select 5 photos from an existing 'Album' to import. Verify all 5 selected photos are successfully imported and displayed within the new 'Project' folder.
**Differentiating Folder Types in Sidebar:** Navigate through the sidebar, visually confirming clear distinction between 'Album' and 'Project' folder types (e.g., by different icons, labels, or styling). Verify that selecting an 'Album' displays photo management UI, and selecting a 'Project' displays modeling data UI.
**Normal Mode Operation:** Upload photos into a new project and initiate the modeling process. Confirm that the process proceeds automatically with default parameters, without any prompts for user parameter adjustment, and completes successfully.
Edge Cases
**SSO Portal Unavailability:** Test system behavior if the external SSO portal is unreachable or returns an error during login attempt.
**Unsupported Photo File Types:** Attempt to upload a file type not typically recognized as a photo (e.g., a document, video, or executable) into an 'Album'.
**Large Number of Photos:** Attempt to upload a very large number of photos (e.g., 1000+) into an 'Album' or import into a 'Project'. Observe performance and system stability.
**Extremely Large Photo File Size:** Upload individual photo files that are exceptionally large (e.g., several hundred MBs or GBs). Monitor upload success, processing time, and potential timeouts.
**Searching in an Empty Album:** Perform a search query within an 'Album' that contains no photos.
**Deleting All Photos from an Album:** Verify the ability to delete every photo from an 'Album', leaving it empty.
**Modeling Process Interruption:** Initiate a modeling process and then simulate a network disconnection or browser closure. Observe the system's state upon reconnection/reopening (understanding resumable uploads are future).
**Creating Project with No Available Photos:** Attempt to create a 'Project' when the user has no photos uploaded in any 'Album' to import.

Risks
High: Reliance on External SSO Portal: Any outages, performance issues, or unforeseen API changes in the external SSO Portal could critically block user access and impact project timelines and user experience for V1. This dependency is fundamental.
Medium: User Frustration with V1 Modeling Limitations: V1 includes 'Output Modeling' but defers 'Expert Mode' (adjustable parameters) and 'AT computation error handling' to future versions. If initial modeling results in 'Normal Mode' frequently require manual adjustment or fail without clear recovery options, users may experience significant frustration, impacting V1 adoption and perceived quality.
Medium: Potential Scope Pressure/Creep for Future Features: A substantial list of advanced features is explicitly deferred to future versions (e.g., project sharing, resumable uploads, advanced storage management, expert mode, AT error handling). While planned, there's a risk of pressure to pull these features into V1 or early iterations if user feedback is strong, potentially impacting V1 delivery focus and quality.
Alignment Gaps
There is a potential expectation gap regarding the 'Output Modeling' feature in V1. While it is a core priority, the deferral of 'Expert Mode' (adjustable parameters) and 'AT computation error handling' might lead stakeholders to expect a more robust or customizable modeling experience than V1's 'Normal Mode' will offer. Clarification on the expected success rate and robustness of 'Normal Mode' output without manual parameter tuning or error recovery is needed.
