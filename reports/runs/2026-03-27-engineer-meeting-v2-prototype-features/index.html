Stakeholder Executive Briefing

Business-focused summary: impact, alignment decisions, and top open questions.

Business Summary

We have confirmed the core features for the initial release, prioritizing photo upload and output modeling. User login will be exclusively managed through our existing external portal, simplifying access and development. We also outlined plans for advanced features to be introduced in future versions.

Key Business Outcomes
Users will have a consistent and familiar login experience using the external portal, without needing new credentials.
The initial product will focus on essential capabilities: uploading photos, organizing them into Albums, and generating modeling outputs.
A clear distinction between personal photo Albums and collaborative Project folders will help users manage their data effectively.
Future updates will introduce valuable features like project sharing, resumable uploads, and advanced modeling controls, enhancing user experience and system capabilities over time.

Quality Assurance

Acceptance criteria, test scenarios, edge cases, and untestable behaviours.

Acceptance Criteria
Given a user attempts to access the system, When the user is redirected to the external SSO portal, Then the user must successfully log in via SSO to access the system.
Given a user attempts to register or log in directly within the system, When the system is accessed, Then no internal login or registration forms are presented.
Given a user is in an Album, When the user initiates a photo upload, Then the photo is successfully added to the Album.
Given a user has photos in an Album, When the user selects a photo and chooses to delete it, Then the photo is permanently removed from the Album.
Given a user has multiple photos in an Album, When the user uses the search functionality, Then photos matching the criteria are displayed.
Given a user creates a new Project, When the user is prompted to add photos, Then the user can select photos from an existing list and import them into the Project.
Given a modeling process is initiated, When the modeling process progresses through stages (e.g., AT computing, point cloud generation, model generation), Then a visual progress bar accurately displays the current stage and overall status.
Given the user is viewing the sidebar, When folder types are displayed, Then 'Album' type folders are visually distinguishable from 'Project' type folders.
Given a user selects an 'Album' folder, When managing content, Then only photo-related data can be stored and managed.
Given a user selects a 'Project' folder, When managing content, Then only modeling-related data can be stored and managed.
Given a user accesses the parameter adjustment interface for modeling, When parameters are presented, Then only 'Normal Mode' default parameters are available, and 'Expert Mode' options are not accessible.

Attempt to upload unsupported file types (e.g., text files, videos) to an Album.
Attempt to upload a very large number of photos to an Album simultaneously.
Search for a photo using an empty search string or special characters.
Attempt to delete a photo that is currently involved in an active modeling process (if applicable).
Initiate a modeling process that completes exceptionally quickly or slowly.
Observe the progress bar's behavior if a modeling stage fails or becomes stuck (before V1 error handling).

Verify 'Expert Mode' settings or interfaces are not accessible through direct URLs or developer tools in V1.
